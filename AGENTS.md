# AGENTS.md

## User Preferences

- Відповідай українською.
- Пиши стисло, технічно, без зайвого наповнення.
- Для багатокрокових або розгалужених задач рухайся поетапно.
- Якщо потрібне рішення користувача, постав одне конкретне питання і дочекайся відповіді.

## Workspace Boundaries

- Перед repo-specific роботою перевіряй `pwd`.
- Не виходь за межі робочої директорії проєкту без явного погодження користувача.
- Не читай і не змінюй файли за межами цього repo, якщо користувач прямо не дозволив це для поточного кроку.
- Не читай приватні ключі, токени, `.env` файли або інші секрети.

## Project Scope

- Проєкт має defensive monitoring scope: акустичне виявлення, оцінка напрямку, confidence, логування та операторські алерти.
- Не додавати функції наведення, ураження, weapon-control або автоматичного бойового застосування.
- Не розширювати польовий/операційний scope без окремого погодження користувача.

## Engineering Rules

- Кожну нову фічу покривати тестами.
- Не видаляти тести.
- Попередні тести змінювати тільки після явного погодження користувача.
- Для Python-змін запускати `.venv/bin/python -m pytest`, якщо залежності доступні.
- Не комітити `.venv`, кеші, generated artifacts, WAV-зразки або секрети.
- Не робити broad refactor без потреби для поточного завдання.

## Git Rules

- Не переписувати історію Git без окремого погодження користувача.
- Не використовувати destructive Git-команди на кшталт `git reset --hard` без прямого запиту користувача.
